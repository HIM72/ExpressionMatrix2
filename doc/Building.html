Building from source

You will need to build the code from source if you want to do one of the following:

  • Run the code on a platform for which a pre-built shared library is not available. Currently, we only provide a pre-built library for Ubuntu 16.04. This will also work on other distributions using similar versions of the Linux kernel, such as Linux Mint 18. However it will not work on other Linux distributions, or on other operating systems such as Windows or MacOS.
  • Modify the code, for example to add functionality, fix bugs, or port it to other systems.
  • Debug the code.

The code requires the following software packages to be available:

  • The gcc compiler. This is installed by default on most Linux distributions. Significant code changes may be required to port the code to systems other than Linux systems.
  • The Boost libraries. Boost version 1.58 is used for Ubuntu (package libbost-all-dev), but the code should not be strictly dependend on a specific version of the Boost libraries.
  • Python3 (on ubuntu 16.04, package python3-all-dev). The code can also be built for Python2 by making appropriate changes in the include files and libraries used in the build process.
  • Graphviz (on ubuntu 16.04, package graphviz).
  • Libraries and development files needed to read HDF5 files (on ubuntu-16.04, packages libhdf5-10, libhdf5-cpp-11, and libhdf5-dev).
The means by which you can obtain and install these packages is dependent on the system you want to run the build the on. The package names given above allow you to download these packages, for ubuntu 16.04, from the standard ubuntu repositories.

Once you have a local copy of the repository, you have two choices to build from source:

  • Use the Eclipse C++ IDE. Eclipse project files .project and .cproject are provided in the top directory of the repository. After opening the project files in Eclipse, you can use Eclipse commands to build the Release or Debug version of the code.
  • Use the makefiles provided in the Release and Debug directories of the repository. Just cd to one of these directory, then run command "make all" to obtain a copy of ExpressionMatrix.so that reflects al the latest changes in the source code.
